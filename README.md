# FrameForge

FrameForge 是一个面向图片与视频素材的高质感展示卡片生成工具。

它不试图成为复杂的设计软件或视频剪辑器，而是让用户上传一张图片或一段视频后，快速得到一张有设计感的展示图、封面图或短视频包装模板。

核心理念：

> 上传素材，自动生成高级展示卡片。

## 产品定位

传统模板工具通常要求用户先浏览大量模板，再上传素材并手动调整。FrameForge 的目标是反过来：

1. 用户先上传素材。
2. 系统根据素材比例、色彩和基础特征自动选择模板。
3. 模板参数在受控范围内生成。
4. 用户得到一个默认就好看的结果。
5. 用户可以继续微调、换一版、导出。

产品重点不是“自由设计”，而是“克制参数 + 高质量模板 + 稳定审美输出”。

## 核心能力

- 图片/视频上传
- 上传即生成的 Magic Mode
- 模板画廊 Gallery Mode
- 参数化模板系统
- 图片裁剪、缩放、定位
- 圆角、边框、阴影、背景、留白、渐隐、文字、署名
- 高清图片导出
- 原清晰度视频导出
- 模板参数保存与分享

## 不做什么

第一阶段明确不做：

- 复杂视频剪辑
- 多轨时间线
- 转场特效
- 粒子特效
- AI 修图
- 完整 Figma 式自由设计
- 海量低质量模板库

## 本地开发

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

类型检查：

```bash
npm run typecheck
```

正式构建：

```bash
npm run build
```

## 当前技术骨架

- Vite
- React
- TypeScript
- 原生 IndexedDB

IndexedDB 当前包含四类本地数据：

- `projects`：项目配置与模板参数
- `mediaAssets`：用户上传的图片/视频 Blob 与基础元数据
- `history`：历史操作记录
- `settings`：本地偏好与上次打开项目

## 当前阶段

项目处于产品定义与技术方案设计阶段。当前目标是先验证：

1. 上传即生成是否足够有吸引力。
2. 少量高质量模板是否能覆盖大量用户素材。
3. 参数化模板是否能稳定输出高级结果。
4. 高清图片与视频导出链路是否可控。

## 推荐 MVP

第一版建议聚焦：

- 空白上传首页
- 图片上传
- Magic Mode 自动生成
- 10 个精选模板
- 每个模板 5-10 个用户可调参数
- 高清 PNG/JPG 导出
- 基础三段式编辑器

视频导出、AI 模板选择、参数分享、模板市场可以作为后续阶段推进。
